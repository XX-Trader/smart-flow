# 简单功能流程

> **适用场景**: 单表操作、纯UI调整、单个接口

---

## 流程图

```
用户输入需求
    ↓
识别为简单功能
    ↓
直接实施代码
    ↓
交付改动摘要
    ↓
用户确认
    ↓
完成 / 快速修复
```

---

## 判定标准

满足以下**任意一个**条件即为简单功能：

- ✅ 只涉及单个接口的增删改查
- ✅ 纯前端UI调整（样式、布局、文案）
- ✅ 修改现有代码的逻辑（如修改判断条件）
- ✅ 添加一个按钮、一个提示、一个校验
- ✅ 不涉及数据库表结构变更
- ✅ 不涉及多个模块联动

---

## 执行原则

### 1. 零确认
不等用户确认，直接按理解实施。如果理解有偏差，用户纠正后再改。

### 2. 快速实施
- 不写需求文档
- 不写技术方案
- 直接改代码

### 3. 交付摘要
改动完成后，提供：
- 改动文件列表
- 关键改动说明
- 需要用户关注的地方

## 示例

### 示例1：添加导出按钮

**用户输入**: "给交易员列表加一个导出按钮"

**我的执行**:
```
1. 直接在 TraderList.vue 中添加导出按钮
2. 实现导出函数（使用 xlsx 库）
3. 交付：
   - 修改了：src/views/pm-robot/TraderList.vue
   - 新增了：exportToExcel() 函数
   - 注意：导出前需要确认导出字段
```

**用户反馈**: "很好" 或 "还要加个导出范围选择"

**我的调整**: 快速修改

---

### 示例2：修改提示文案

**用户输入**: "把'创建成功'改成'创建成功啦'"

**我的执行**:
```
1. 搜索所有"创建成功"
2. 批量替换
3. 交付：修改了3个文件
```

---

### 示例3：添加字段校验

**用户输入**: "分组名称不能超过50个字符"

**我的执行**:
```
1. 在前端表单添加校验规则
2. 在后端API添加校验逻辑
3. 交付：
   - 前端：GroupCreateDialog.vue
   - 后端：views.py 中的 create() 函数
```
