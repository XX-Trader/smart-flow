# 需求关键程度评分系统

> **版本**: v2.1.0
> **创建日期**: 2026-01-07
> **用途**: Smart Flow 功能关键程度评估
> **原则**: 简单实用，避免过度复杂

---

## 📊 评分维度（100分制）

### 1. 业务价值（40分）
- **核心功能**: 40分 - 如果没有这个功能，系统无法运行
- **重要功能**: 25分 - 显著影响用户体验
- **辅助功能**: 10分 - 提升体验，但非必须
- **可选功能**: 5分 - 锦上添花

### 2. 优先级（30分）
- **P0（必须）**: 30分 - MVP 必须实现
- **P1（重要）**: 20分 - 第一期迭代
- **P2（可选）**: 10分 - 后续考虑

### 3. 实现难度（30分）
- **简单**: 30分 - 1-2天完成，风险低
- **中等**: 20分 - 3-5天完成，有一定风险
- **复杂**: 10分 - 1周以上，风险高

## 🔄 评分流程

### Step 1: 识别功能
```yaml
功能名称: [功能名]
功能描述: [简短描述]
```

### Step 2: 快速评估
```yaml
业务价值: [40/25/10/5]
优先级: [30/20/10]
实现难度: [30/20/10]
```

### Step 3: 计算总分
```python
总分 = 业务价值 + 优先级 + 实现难度
```

### Step 4: 分类
- **90-100分**: 核心关键功能，立即实现
- **60-89分**: 重要功能，第一期内实现
- **30-59分**: 辅助功能，第二期考虑
- **<30分**: 可选功能，有时间再做
