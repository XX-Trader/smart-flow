```markdown

## 1. 任务概要

## 4. 行动建议
```

## 3. 自动化测试代码

### 3.1 单元测试

```javascript
// [文件名].test.js
describe('[功能模块名称]', () => {
  beforeEach(() => {
    // 测试前准备
  });

  describe('[功能点]', () => {
    it('should [预期行为]', () => {
      // Arrange
      const input = '...';

      // Act
      const result = functionToTest(input);

      // Assert
      expect(result).toBe('...');
    });

    it('should handle [边界条件]', () => {
      // 测试代码
    });
  });

  afterEach(() => {
    // 清理
  });
});
```

### 3.2 集成测试

```javascript
describe('[模块集成测试]', () => {
  it('should [集成场景]', async () => {
    // 测试完整流程
  });
});
```

### 3.3 端到端测试

```javascript
// E2E 测试示例
describe('[用户场景]', () => {
  it('should complete [用户流程]', async () => {
    await page.goto('/path');
    await page.fill('#input', 'value');
    await page.click('#button');
    await expect(page).toHaveURL(/success/);
  });
});
```

### 3.4 Mock 配置

```javascript
// Mock 外部依赖
jest.mock('[模块名]', () => ({
  [函数名]: jest.fn(() => [返回值])
}));
```

## 📊 模板使用指南

### 选择合适的模板

1. **test-automator**: 需要编写自动化测试时
2. **code-reviewer**: 代码审查 PR 时
3. **security-auditor**: 全面安全审计时
4. **api-security-audit**: API 专项安全审计时
5. **test-driven-development**: TDD 开发流程
6. **verification-before-completion**: 任务完成前验证
7. **debugger**: Bug 调试分析
8. **error-detective**: 日志错误分析

### 模板定制

根据项目具体需求，可以:
- 删除不适用的章节
- 增加项目特定的检查项
- 调整评分标准
- 修改输出格式

### 最佳实践

1. **完整输出**: 确保所有相关章节都有内容
2. **具体明确**: 避免模糊描述，提供具体示例
3. **可操作性**: 建议应该具体、可执行
4. **优先级清晰**: 明确标记必须修复 vs 建议改进
5. **跟踪状态**: 使用 ✅ ❌ 等符号标记检查项
